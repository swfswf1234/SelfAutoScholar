## 核心模块说明

### 1. Discovery Agent — 数据发现

负责从各数据源抓取候选资料：

| 数据源 | 客户端 | 输出 |
|--------|--------|------|
| arXiv | `arxiv` 库 | 论文元数据 (标题、摘要、作者、分类) |
| GitHub | `PyGithub` 库 | 项目元数据 (名称、描述、README、Stars) |
| RSS/新闻 | `feedparser` 库 | 新闻元数据 (标题、内容、来源) |
| MCP 增强 | `mcp` SDK | 补充搜索结果 |

### 2. Dedup Filter — 去重过滤

**规则：精确标题匹配**

```python
def is_duplicate(title: str, item_type: str) -> bool:
    """检查标题是否已存在于数据库中"""
    if item_type == "paper":
        return db.query(Paper).filter(Paper.title == title).first() is not None
    elif item_type == "project":
        return db.query(Project).filter(Project.github_id == github_id).first() is not None
    elif item_type == "news":
        return db.query(News).filter(News.title == title).first() is not None
    return False
```

### 3. Evaluator — 智能评估

调用外部 LLM API，对每条资料输出三个二元标签：

```json
{
  "importance": "important",      // important | not_important
  "relevance": "relevant",        // relevant | not_relevant
  "interest": "interested"        // interested | not_interested
}
```

### 4. Download Decision — 下载决策

**决策矩阵：**

| 重要性 | 相关性 | 兴趣 | 下载? |
|--------|--------|------|-------|
| 重要 | 相关 | 感兴趣 | ✓ |
| 重要 | 相关 | 不感兴趣 | ✓ |
| 重要 | 不相关 | 感兴趣 | ✓ |
| 重要 | 不相关 | 不感兴趣 | ✗ |
| 不重要 | * | * | ✗ |

**简化规则：重要 + (相关 OR 感兴趣) → 下载**

### 5. Analysis Agent — 深度分析

```
论文 PDF → pymupdf4llm → Markdown → 本地 LLM → 结构化摘要
项目 README → 本地 LLM → 功能分析 + 中文翻译
新闻全文 → 本地 LLM → 要点提取 + 中文翻译
```

### 6. Interest Agent — 兴趣迭代

根据用户打标记录更新兴趣画像：

```
用户标注 "感兴趣" → 增加相关领域/关键词权重
用户标注 "不感兴趣" → 降低相关领域/关键词权重
```

## 配置项

`setting.ini` 中的相关配置：

```ini
[Discovery]
max_candidates = 100          # 每日最大候选资料数
max_downloads = 10            # 每日最大下载数 (Top N)

[Scheduler]
enabled = true
cron = 0 0 * * *              # 每日 00:00 执行
timezone = Asia/Shanghai

[LLM]
external_api_base = https://api.openai.com/v1
external_api_key = sk-xxx
external_model = gpt-4o
local_api_base = http://127.0.0.1:8000
local_model = default

[Paths]
downloads_dir = data/downloads
exports_dir = exports/markdown
```
